# 个人博客网址上线

博客终于上线了。

地址：[https://lastdba.com](https://lastdba.com)

其实博客一直都有，之前是用 Jekyll + GitHub Pages 搭的，域名是 `liuzhilong62.github.io/blogs`。76 篇文章，都是这几年写 PostgreSQL 的东西——案例、内功、源码、论文精读。

这次算是正式「上线」：换框架、换域名、换主题，从头到尾搞了一遍。

## 换了什么

**框架：Jekyll → Hugo**

Jekyll 其实挺好用，但本地预览一直跑不起来——macOS 上 Ruby 版本太老，`bundle exec jekyll serve` 直接报错。每次改个 CSS 都要 push 到 GitHub 等 Actions 跑完才能看到效果，体验太差了。

Hugo 就不一样。`brew install hugo` 一个二进制文件搞定，76 篇文章构建只要 200 毫秒。本地 `hugo server` 秒开，改完即时刷新。

**主题：minima → Blowfish**

原来是 Jekyll 自带的 minima，自己写了 2000 行 CSS 硬改。TOC 侧边栏、暗色模式、搜索、代码复制——全是手写的 JS。

换了 Hugo 之后选了 Blowfish 主题，上面这些功能全部内置。配置几行 YAML 全搞定，CSS 从 2000 行变成 0 行。

**域名：github.io → lastdba.com**

买了 `lastdba.com`，配了 Cloudflare。GitHub Pages 绑自定义域名，免费 HTTPS 证书，全自动续。

**图片本地化**

原来文章里的图片散落在各个地方——CSDN 的 CDN、GitHub PicBed、墨天轮 OSS。CSDN 有防盗链，换了域名之后图片全部裂开。

写了个脚本把 348 张图全部下载到仓库的 `static/img/` 里，文章里的链接全替换成本地路径。以后再也不怕图床跑路。

## 上线感悟

整个过程折腾了大概一天。

说实话，换 Hugo 最大的感受不是技术上的，而是「不要重新发明轮子」。之前花了很多时间手写暗色模式、TOC、搜索，结果发现换个主题全自带，写得还不如人家好看。

另外，`lastdba.com` 这个域名挂上去之后，博客突然有了一种「正式感」。以前 `liuzhilong62.github.io/blogs` 像是个人的实验项目，现在像一个真正的网站。虽然内容没变，但感觉不一样了。
